# 怖客 - 全简中互联网恐怖灵异内容聚合平台

## 一、项目概述

### 1.1 项目定位
怖客是一款专注于全简中互联网恐怖灵异内容聚合的网页应用，通过爬虫技术收集相关博客评论区数据，筛选用户推荐度高的故事内容进行整合展示，为恐怖灵异爱好者提供优质内容发现平台。

### 1.2 MVP目标 (V1版本)
- 覆盖B站博主"大佬何金银"最新更新的50期内容
- 从50期内容中整理出前10条被"求出处，求推荐"提及次数最多的故事
- 实现完整的浏览、筛选、跳转功能

### 1.3 数据源
- 主数据源：B站博主"大佬何金银"
- 空间地址：https://space.bilibili.com/28346875
- 内容类型：《道听途说》系列灵异故事

## 四、数据模型设计

### 4.1 核心数据表

```sql
-- 视频信息表
CREATE TABLE videos (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  bvid VARCHAR(20) UNIQUE NOT NULL COMMENT 'B站视频BV号',
  aid BIGINT COMMENT 'B站视频AV号',
  title VARCHAR(500) NOT NULL COMMENT '视频标题',
  description TEXT COMMENT '视频简介',
  cover_url VARCHAR(500) COMMENT '封面图URL',
  duration INT COMMENT '视频时长(秒)',
  pub_date DATETIME COMMENT '发布时间',
  view_count INT DEFAULT 0 COMMENT '播放量',
  danmaku_count INT DEFAULT 0 COMMENT '弹幕数',
  comment_count INT DEFAULT 0 COMMENT '评论数',
  series_name VARCHAR(100) COMMENT '系列名称',
  episode_number INT COMMENT '期数',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_pub_date (pub_date),
  INDEX idx_series (series_name, episode_number)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 故事信息表
CREATE TABLE stories (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  video_id BIGINT NOT NULL COMMENT '关联视频ID',
  title VARCHAR(500) COMMENT '故事标题',
  summary TEXT COMMENT '故事摘要',
  start_time INT COMMENT '故事开始时间(秒)',
  end_time INT COMMENT '故事结束时间(秒)',
  part_number INT DEFAULT 1 COMMENT '分P号',
  heat_score DECIMAL(10,2) DEFAULT 0 COMMENT '热度分数',
  mention_count INT DEFAULT 0 COMMENT '提及次数',
  like_count INT DEFAULT 0 COMMENT '点赞总数',
  ai_comment TEXT COMMENT 'AI生成的简评',
  tags JSON COMMENT '标签数组',
  status TINYINT DEFAULT 1 COMMENT '状态: 1-正常 0-下架',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (video_id) REFERENCES videos(id),
  INDEX idx_heat (heat_score DESC),
  INDEX idx_mention (mention_count DESC)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 评论数据表
CREATE TABLE comments (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  rpid BIGINT UNIQUE NOT NULL COMMENT 'B站评论ID',
  video_id BIGINT NOT NULL COMMENT '关联视频ID',
  story_id BIGINT COMMENT '关联故事ID',
  parent_id BIGINT COMMENT '父评论ID',
  member_mid BIGINT COMMENT '用户ID',
  member_name VARCHAR(100) COMMENT '用户名',
  content TEXT NOT NULL COMMENT '评论内容',
  like_count INT DEFAULT 0 COMMENT '点赞数',
  reply_count INT DEFAULT 0 COMMENT '回复数',
  ctime DATETIME COMMENT '评论时间',
  is_seek_source TINYINT DEFAULT 0 COMMENT '是否求出处',
  is_seek_recommend TINYINT DEFAULT 0 COMMENT '是否求推荐',
  mentioned_story VARCHAR(500) COMMENT '提及的故事',
  mentioned_time VARCHAR(50) COMMENT '提及的时间点',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (video_id) REFERENCES videos(id),
  INDEX idx_video (video_id),
  INDEX idx_seek (is_seek_source, is_seek_recommend)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 故事提及关联表
CREATE TABLE story_mentions (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  story_id BIGINT NOT NULL,
  comment_id BIGINT NOT NULL,
  mention_type ENUM('source', 'recommend', 'time') COMMENT '提及类型',
  confidence DECIMAL(3,2) COMMENT '置信度',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (story_id) REFERENCES stories(id),
  FOREIGN KEY (comment_id) REFERENCES comments(id),
  UNIQUE KEY uk_story_comment (story_id, comment_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 十一、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| B站反爬机制 | 数据采集失败 | 降低频率、模拟用户行为、代理池 |
| 评论数据量大 | 处理性能问题 | 分批处理、增量更新 |
| AI简评质量 | 用户体验差 | 人工审核、模板优化 |
| 视频链接失效 | 跳转失败 | 定期检查、备用链接 |
| 版权问题 | 法律风险 | 仅做聚合跳转、不存储原内容 |
